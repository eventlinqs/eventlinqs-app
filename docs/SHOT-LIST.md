# Photo shopping list (shot list)

Your complete buy list for photo day. Every row is one photo to download. Save
each download under the **exact filename** in column 2, drop it into
`design-assets/incoming/`, then run `node scripts/ingest-imagery.mjs` (see
`docs/PHOTO-DAY.md`). The filename is the wiring, so it has to be exact.

**110 slots**, built only from the real platform taxonomy: the 8 real
`event_categories`, the 19 locked Scene-layer scenes, the 13 Australian launch
cities, plus homepage and organiser heroes and venue shots.

## Where to shop

- **SNIPER (25 slots): buy on Adobe Stock.** These are the showstoppers people
  judge the brand by: homepage hero, organisers hero, city heroes, marquee
  event covers. Spend the premium budget here.
- **VOLUME (85 slots): buy on Envato.** Everything else. Good, on-brand, cheap
  per image at volume.

## The five laws for every photo

1. **Landscape orientation.** Always. No portrait, no square.
2. **Mix the light.** Across each group, rotate bright daytime, golden hour, and
   night. Never a dark monotone set. Each row tells you which light to buy.
3. **Faces enjoying.** Real people, visibly having a good time. Not empty rooms
   (venues are the only exception, and even those should feel alive).
4. **Australian-looking where possible.** AU cities, AU light, multicultural
   crowds that look like a real Australian audience.
5. **Minimum size.** Hero 1920px wide or larger, category and scene 1440px or
   larger, city and venue 1200px or larger. Landscape. The ingest script
   rejects anything smaller, so check before you buy.

## Filename convention (do not improvise)

`role__key__city__descriptor.jpg` with **double** underscores between fields.
Use `none` for a field that does not apply. The filenames below are already
correct: copy them exactly.

## After you have bought them

1. Put every file in `design-assets/incoming/` with the exact name from column 2.
2. Run `node scripts/ingest-imagery.mjs --dry-run` first. It checks names, sizes,
   and light. Fix anything it flags.
3. Run `node scripts/ingest-imagery.mjs` to optimise and upload. Done.

The taxonomy in this list is locked to the real platform: categories from
`event_categories`, scenes from the Scene layer, cities from the launch list. If
the taxonomy changes, regenerate this list rather than editing slugs by hand.
