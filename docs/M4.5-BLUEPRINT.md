## Critical Items - Blocking Launch Quality

> These items MUST be fixed before EventLinqs is shown to any real user, investor, or press without embarrassment. Every item below would cause a first-time visitor to leave immediately.

---

### P1-1: Filter Sheet Auto-Opens on Mobile Events Page (CRASH-LEVEL BUG)
**Observed:** On every navigation to `/events` at 375px viewport, the filter bottom-sheet renders in the open state. The close button is rendered outside the visible viewport (element outside viewport error confirmed in Playwright). The user cannot dismiss it. The events list is partially obscured.

**Root cause to investigate:** The filter component is initialising `isOpen` state to `true` instead of `false`, OR a `useEffect` is triggering an open on mount at mobile widths.

**Ticketmaster:** Filter chip bar is always visible, never auto-opens a sheet.
**Eventbrite:** Filter chips are visible; "More filters" button opens modal only on click.
**DICE:** Genre chips shown, no auto-open.

**Excellence target:** Filter sheet is closed on load; opens only on explicit "Filters" button tap.
**Estimate:** 1-2h

---

### P1-2: Homepage Is an Empty Placeholder
**Observed:** The homepage is a full-screen grey rectangle with a centred headline and two buttons. There are no featured events, no event categories, no social proof numbers, no imagery, no "trending near you" section, no footer links. It looks like a mockup wireframe, not a product.

**Ticketmaster:** Full-width hero video/image of concert, featured tours above the fold, city-based recommendations, category navigation strip.
**Eventbrite:** Search bar front and centre, "Events near [City]" personalisation, category grid, curated "Top picks" section.
**DICE:** Artist photography hero, genre carousels, "What's On in [City]" personalisation.

**What needs to exist above the fold:**
1. Hero section - large background image or gradient with search bar + location. Pull from featured/upcoming events.
2. "Upcoming Events Near You" section - first 6 real events from DB as cards (same card component as `/events`)
3. Category icon grid - 8 categories as tappable chips with icons
4. Footer - links to About, For Organisers, Privacy, Terms + social icons

**Excellence target:** A visitor landing on the homepage immediately sees events, understands the value proposition, and can begin discovery without clicking "Browse Events."
**Estimate:** 6-8h (homepage rebuild is the single biggest M4.5 task)

---

### P1-3: Event Detail Page Is a Text Dump
**Observed:** The event detail page shows: category badge, title, date/time, "Australia" as location, organiser name, "Copy link", then a ticket widget. That is the entire page. There is no hero image area, no event description, no map, no proper venue display, no WhatsApp share, no Add to Calendar, no social proof, no similar events.

**Ticketmaster:** Full-width event hero (2:1 aspect), event title + date in overlay, sticky ticket CTA, venue map, artist/event description, setlist section, related events.
**Eventbrite:** 16:9 event cover image at top, full event description (rich text), venue map, organiser card with follow button, "You might also like" section, sticky "Register" CTA at top.
**DICE:** Artist/event photo full-bleed hero, event description, venue info with map link, line-up list, share to friends (WhatsApp prominent).

**What needs to exist:**
1. Hero image area - 16:9 cover image with `object-cover`. When no image: gradient placeholder using brand colours, never a blank white box.
2. Event description - rich text field displayed in the body. Currently this field exists in the DB but is not rendered on the public page.
3. Venue display - full address (not just country), link to Google Maps.
4. Share buttons - Copy link ✓ already exists. Add: WhatsApp, Twitter/X.
5. Add to Calendar - Google Calendar + Apple .ics download links.
6. Sticky bottom CTA on mobile - "Get Tickets" button fixed to bottom of screen. Currently it scrolls off.
7. Organiser section - avatar, name, "View other events by [Organiser]" link.

**Excellence target:** An attendee landing on an event page immediately knows what it is (image), why they should go (description), where it is (venue + map), and can buy in one tap (sticky CTA).
**Estimate:** 6-8h

---

### P1-4: "Select Tickets" Is Not Sticky on Mobile

## Deployment Readiness Checklist

These 5 tracked items from Phase 3 verification must be actioned before production traffic:

- [ ] **`QUEUE_SECRET` env var set in Vercel production** - if not set, HMAC tokens fall back to hardcoded `'dev-queue-secret-change-in-prod'`. Every user in the queue would be vulnerable to token forgery. Set in Vercel Dashboard → Settings → Environment Variables → Production.
- [ ] **`queue-admit` cron wired in `vercel.json`** - the route `/api/cron/queue-admit` exists and returns 200, but Vercel will never call it unless registered. Add to `vercel.json` crons array: `{ "path": "/api/cron/queue-admit", "schedule": "* * * * *" }` (every minute).
- [ ] **`waitlist-expire` cron wired in `vercel.json`** - same issue. Should run every 5 minutes: `"schedule": "*/5 * * * *"`.
- [ ] **`squad-expire` cron wired in `vercel.json`** - same. Every 5 minutes.
- [ ] **`expire_stale_squads_return_type` migration applied to production** - migration `20260412000001` was applied to dev branch. Confirm it is applied to the production Supabase project before enabling squad booking publicly.

## Tier 2 - Surpass the Competitors (must-have competitive edge)

These items take EventLinqs from "meets baseline" to "beats the competition." Tier 1 (the existing P1 items) fixes amateur elements; Tier 2 creates the reasons someone chooses EventLinqs over TM/EB/DICE.

1. **Transparent pricing banner on checkout** - HIGH - direct attack on Ticketmaster's hidden-fee weakness. Show "Zero hidden fees. What you see is what you pay." prominently.
2. **"Who's going" social layer** - HIGH - avatars + count on event pages. DICE has light version, TM doesn't at all. Creates social proof that drives conversions.
3. **PWA with offline ticket viewing + push notifications** - HIGH - all 3 competitors fail here. Installable, works without signal at the venue.
4. **WhatsApp share-first UX** - HIGH - not generic Share API. African diaspora killer feature. Direct wa.me deep links with pre-filled messages.
5. **Organiser dashboard with genuinely beautiful charts** - HIGH - not merely functional. Every competitor has data; nobody makes it impressive. Use Recharts with custom styling, smooth animations.
6. **Real-time viewer count + seats-selling indicator** - MEDIUM - "47 people viewing this event, 3 tickets sold in the last 5 minutes." Creates better urgency than static "X tickets left" banners.
7. **Native Apple Wallet + Google Wallet pass integration** - MEDIUM - DICE does this, TM partially, EB poorly. Tickets that live natively in the phone's wallet.
8. **One-tap refund flow with transparent policy** - MEDIUM - direct attack on Eventbrite's worst UX. No support tickets, no "contact us" dead ends. Self-serve refund within published policy window.

Each Tier 2 item must be benchmarked against all 3 platforms (Rule A) before being considered complete. The standard is "exceeds all 3" not "matches any."
